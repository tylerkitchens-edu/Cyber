Level 3: Networking

Wireless Networking Basics

Wireless networking has transformed how devices connect and communicate, eliminating the need for physical cabling and providing flexibility for users and organizations alike. From laptops and smartphones to IoT devices, wireless networks support a wide range of technologies such as Wi-Fi, Bluetooth, cellular (LTE/5G), and NFC. Each of these technologies uses radio frequency (RF) signals to transmit data through the air.

Most wireless networks operate on 2.4 GHz or 5 GHz frequency bands. The 2.4 GHz band provides greater range but is more prone to interference (from microwaves, cordless phones, etc.), while 5 GHz offers faster speeds and less congestion but a shorter range. Modern networks are increasingly adopting 6 GHz Wi-Fi 6E for improved bandwidth and reduced interference.

Understanding wireless performance involves analyzing factors such as signal strength (RSSI), signal-to-noise ratio (SNR), and channel overlap. These determine the reliability and speed of data transmission. Technicians often use tools like Wi-Fi analyzers to optimize access point placement and minimize interference.

Wireless Security

Because wireless transmissions travel through open air, they are inherently more vulnerable to eavesdropping and unauthorized access than wired networks. Implementing robust encryption and authentication standards is critical. Modern networks should use WPA3 (Wi-Fi Protected Access 3) for the highest level of security. Avoid outdated protocols like WEP and WPA2-PSK, which are easily compromised.

Other best practices include disabling SSID broadcasting for private networks, using MAC address filtering, and segmenting guest traffic onto separate VLANs to prevent internal network exposure.

Introduction to VLANs and VPNs

Virtual Local Area Networks (VLANs) and Virtual Private Networks (VPNs) are technologies designed to enhance network efficiency, control, and security.

VLANs

A VLAN allows administrators to logically segment a single physical network into multiple virtual networks. For example, a company can separate HR, Finance, and IT traffic, even if they share the same physical switches. This segmentation reduces broadcast domains, enhances performance, and isolates sensitive data to improve security. VLANs are configured on managed switches using IEEE 802.1Q tagging standards.

VPNs

VPNs extend private networks securely across public infrastructure, such as the internet. They use encryption protocols (like IPsec or SSL/TLS) to ensure data confidentiality and integrity during transmission. VPNs are essential for remote workers who need secure access to internal company systems from outside locations. Common types include site-to-site VPNs for linking branch offices and remote-access VPNs for individual users.

Together, VLANs and VPNs give organizations the flexibility to maintain security and scalability across diverse network environments.

Network Security Essentials

Importance of Network Security

As networks expand and connect more devices, the risk of unauthorized access, data breaches, and malware infections increases. Network security aims to protect data in transit and ensure the integrity and availability of critical systems. Security starts with layered defenses—often referred to as defense-in-depth—combining multiple technologies and best practices to create redundancy in protection.

Key Security Measures

  • Firewalls: Monitor and control incoming and outgoing traffic based on predefined security rules.
  • Intrusion Detection and Prevention Systems (IDS/IPS): Detect suspicious network activity and block potential attacks.
  • Encryption: Protects sensitive information as it moves across networks (e.g., HTTPS, IPsec).
  • Multi-Factor Authentication (MFA): Requires multiple verification factors for secure access beyond just passwords.
  • Security Audits: Regular evaluations that ensure compliance with organizational policies and industry standards.

Emerging Threats

Modern organizations face evolving threats like ransomware, phishing, zero-day exploits, and social engineering attacks. Security awareness training is one of the most effective countermeasures—educating employees to identify suspicious links, attachments, and login attempts helps prevent incidents before they escalate.

Common Network Ports

Network ports are logical endpoints used by applications to identify specific processes or network services. They operate at the Transport Layer (Layer 4) of the OSI model, working primarily through TCP or UDP protocols. Understanding common ports helps in configuring firewalls, diagnosing connectivity issues, and recognizing potential threats.

  • Port 80 / 443: Used for HTTP and HTTPS web traffic. HTTPS (443) provides encryption for secure browsing.
  • Port 53: Domain Name System (DNS) — translates human-readable domain names into IP addresses.
  • Port 123: Network Time Protocol (NTP) — synchronizes system clocks across devices to maintain authentication and logging accuracy.
  • Ports 67 (TCP) / 68 (UDP): Dynamic Host Configuration Protocol (DHCP) — automates IP address assignment between clients and servers.
  • Port 3389: Remote Desktop Protocol (RDP) — enables remote control of Windows computers and servers.

Technicians and administrators should memorize these and other key ports, as they’re commonly encountered during troubleshooting, configuration, and security assessments.
