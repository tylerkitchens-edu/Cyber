Storage Types

A processor uses random access memory (RAM) to store information about a program it's currently running. RAM is extremely fast, and the processor can read from and write to it billions of times each second. However, RAM is volatile; if the system shuts off, RAM loses everything it had stored in memory. So, we have Mass storage devices to store important data for persistent use.

This lesson covers the topic of common Mass storage devices.

Common Storage Devices

To store information when the computer is turned off (called persistent storage), computers must use long-term storage. Otherwise, data would be lost whenever the system shuts down. There are several different types of long-term storage devices that are commonly implemented in most computer systems.

  • SCSI - A SCSI (scuzzy) controller was one of the first disk controllers. It was developed in the 1970s. It has a 50-pin flat ribbon connector. SCSI controllers are now being phased out.
  • IDE/EIDE/PATA - Controllers based on the IBM PC industry standard architecture (ISA) 16-bit bus standard.
  • SATA - SATA supports disk hot-swapping. This means the drive can be changed while the computer is powered on. SATA uses better cables and provides faster speeds than SCSI and IDE/EIDE/PATA.
  • M.2 - M.2 is a compact form factor for solid-state storage devices that has become the standard for modern PCs and laptops. Unlike traditional 2.5-inch SATA drives, M.2 drives are small rectangular cards that plug directly into the motherboard, saving space and eliminating cable clutter. They come in various lengths (commonly 2280, meaning 22mm wide by 80mm long) and can use either the SATA interface or the much faster NVMe protocol over PCIe lanes. NVMe M.2 drives offer significantly faster read/write speeds—often exceeding 5,000 MB/s or even 7,000 MB/s with PCIe 4.0 and 5.0—compared to SATA's 550 MB/s limit.

SATA is (or was) the most common controller used in desktop and laptop computers. There are usually four SATA ports on most motherboards. Nowadays modern systems are moving towards M.2 similarly to how UEFI is replacing BIOS

A typical desktop computer system has several storage devices attached to it. For example, it may have a solid-state drive (SSD) for the operating system and key programs, a hard disk drive (HDD) for file storage, an optical drive for CDs and DVDs, and removable USB storage devices.

Drives

Typically, each storage device on the computer is called a drive. A drive is the top-level folder, or directory, of a file system. Each drive is assigned a letter by the computer. The one that contains the operating system is typically called the C drive (C:).

What happened to the A and B drives?In the early days of desktop computers, the A and B drives were typically used for 3/5-inch and 5-inch floppy disks, respectively. Even though floppy disks have been surpassed by better technology, the tradition of calling the principal hard drive the C drive is still with us.

Partitioning

Most operating systems in the 1990s could only support drives of a certain size. For example, Windows 95 only allowed a maximum drive size of 32GB. This was a problem if someone wanted to use a hard drive with more capacity than this. The solution to this problem was partitioning, or dividing a single physical disk into multiple logical drives called volumes. Typically, one of these volumes would be the C drive and contain the operating system, and the others would be assigned other drive letters.

Today's operating systems can support enormous drive sizes. Because of this, partitioning isn't as common as it used to be. However, you still may want to partition a disk if you want to:

  • Dual boot your computer with two operating systems, such as Windows and Linux
  • Hide sensitive files on a partition without an assigned drive letter
  • Back up files that may be lost if the operating system becomes corrupted
  • Save an image of the operating system on the same disk as the operating system itself

Each volume you create on a disk consumes some of the storage space. In other words, if you create three volumes on a 500GB HDD, you may be left with only 400 GB of available storage space.
