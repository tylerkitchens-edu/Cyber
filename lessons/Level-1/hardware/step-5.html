Boot Process

Lesson

Learning Objectives

  • Understand the sequence of events during the boot process, from power-on to loading the operating system
  • Differentiate between BIOS and UEFI, and understand their roles in system initialization
  • Explain the role of CMOS in storing hardware settings and boot configuration

Boot Process / Digital Data

Every computer follows a structured process when starting up, known as the boot process. In this section, we will explore how a system initializes hardware, loads firmware, and eventually boots into an operating system. Additionally, we will examine how digital data is processed, stored, and transferred within a computer system. Understanding these concepts helps in troubleshooting boot issues and understanding binary data handling.

  1. Step 1: Activate BIOS - Electrical power activates the Basic Input/Output System (BIOS).
  2. Step 2: Check CMOS - The BIOS reads the CMOS chip for system settings and instructions for hardware initialization.
  3. Step 3: Initialize Hardware - The BIOS wakes up and configures hardware devices such as fans, memory, hard drives, and processors.
  4. Step 4: POST - BIOS performs a Power-On Self-Test (POST) to verify that all hardware is functioning properly.
  5. Step 5: Find an OS - BIOS searches for a bootable device according to the boot order stored in the CMOS.
  6. Step 6: Load OS into RAM - BIOS loads the operating system into RAM, and the processor begins executing OS instructions. BIOS then becomes dormant.

BIOS & UEFI

Accessing BIOS usually requires pressing a key (F1, F2, Del, etc.) during boot. Within BIOS settings, you can adjust CPU clock, memory timings, and boot order.

BIOS: Non-volatile firmware stored on a memory chip on the motherboard. Runs first and loads the OS. Settings are retained via CMOS.

UEFI: Modern graphical boot tool with advantages including faster booting, support for large drives, enhanced security, and EFI file-based startup. Provides an easier GUI for configuration.

CMOS: A small RAM chip powered by a battery that stores BIOS settings such as system date, time, and hardware configuration.

BIOS is software; CMOS is hardware. BIOS uses CMOS to save critical system information, even when power is disconnected.
