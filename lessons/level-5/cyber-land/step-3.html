Lesson: Security Architecture and Security Zones

Network Security

Learning Objectives

  • Define network security and explain its importance in protecting sensitive data and resources.
  • Describe fundamental concepts of network architecture, including segmentation and redundancy strategies.
  • Identify and explain the purpose of security zones such as intranet, extranet, DMZ, honeynet, and ad hoc networks.
  • Understand and apply identity and access management principles, including authentication and role-based access control.
  • Implement essential security practices, including defense in depth, network monitoring, and regular security audits.

Network Security and Architecture

Network Security: Network security protects systems, data, and resources from unauthorized access, cyberattacks, and other threats. Common tools and practices include firewalls, intrusion detection/prevention systems (IDS/IPS), encryption protocols, and continuous monitoring.

Network Architecture: Well-designed network architecture ensures efficient data flow, resource availability, and minimal vulnerabilities. Implementing segmentation, redundancy, and layered security (defense in depth) improves resilience and reduces risk exposure.

Essential Security Practices:

  • Defense in Depth: Use multiple layers of security instead of relying on a single protective measure.
  • Regular Security Audits: Continuously monitor and test network security to identify weaknesses.
  • Access Control: Enforce authentication and authorization to restrict access to sensitive systems.
  • Network Monitoring: Track traffic and detect anomalies or suspicious activity.

Security Zones

Security zones segment networks based on risk, access, or sensitivity of the resources they protect. They allow organizations to apply tailored security measures for each zone.

Security Zone Networks

  • Honeynet: A controlled network environment designed to attract and analyze attacker activity without compromising production systems.
  • Ad Hoc Network: A temporary, decentralized network that allows direct device-to-device communication without a traditional base station or router.

Intranet · Extranet · DMZ — In-depth

Definitions, architecture, security implications, examples, and best practices.

1. Intranet — Internal Private Network

Definition: A private network accessible only to an organization's internal users (employees, internal systems). Used for internal communication, collaboration, and hosting sensitive or proprietary internal applications.

Purpose

  • Internal collaboration (wikis, HR portals, ticketing).
  • Host internal business applications (payroll, inventory).
  • Protect proprietary data and internal workflows.

Access & Security

  • Restricted to internal users — often accessible only via corporate LAN or VPN.
  • Authentication typically via Active Directory/LDAP and MFA.
  • Protected by segmentation, firewalls, and monitoring (SIEM).

Technologies

AuthActive Directory / LDAP, SSO

NetworkInternal DNS, VLANs, VPN gateways

// Example flow
Employee → Corporate LAN → Intranet server (HR portal, wiki)

2. Extranet — Controlled External Access

Definition: A controlled extension of the intranet that gives authorized external parties (partners, vendors, customers) limited and audited access to specific internal resources.

Purpose

  • Enable B2B data sharing and collaboration (partner portals, vendor dashboards).
  • Expose limited APIs or document shares to trusted third parties.

Access & Security

  • External access must be authenticated and authorized (SSO, MFA, client certs).
  • Often sits behind reverse proxies, WAFs, and gateway controls.
  • Strict logging, encryption, and least-privilege controls are mandatory.

Technologies

SecurityWeb Application Firewall (WAF), Reverse proxy

AuthSAML/OAuth, client certs, MFA, VPN

// Example flow
Partner → Secure Portal / VPN → Extranet server → Controlled access to specific internal resources

3. DMZ — Demilitarized Zone

Definition: A segmented subnetwork between an organization's internal network and the internet. It hosts public-facing services (web, mail, DNS) and acts as a buffer to protect the internal network.

Purpose

  • Isolate public-facing services from the internal network.
  • Limit attack surface and contain breaches.
  • Control and inspect traffic crossing between internet and intranet.

Security Controls

  • Two-firewall design or multi-interface firewall (policy separation).
  • Restrict DMZ → internal traffic to minimal, specific flows (e.g., app → DB port only).
  • Use IDS/IPS, strict ACLs, and logging for all DMZ hosts.

Security Best Practices

  • Network segmentation: Use VLANs & firewalls to isolate zones.
  • Zero trust: Authenticate and authorize each request, even internal ones.
  • Least privilege: Only open required ports and services between zones.
  • Monitoring: Centralized logging + SIEM for DMZ and extranet access.
  • Encryption: TLS for all external or partner-facing channels.
  • Patch & harden: Keep DMZ hosts patched and minimize attack surface (disable unused services).

Identity and Access Management

Identity Management: Processes that identify users, devices, or systems, and manage their credentials to ensure secure access. Proper identity management prevents unauthorized users from accessing sensitive resources.

Access Control: Enforces permissions and restrictions using techniques such as role-based access control (RBAC). By granting access only to authorized users, organizations reduce the risk of internal and external security breaches.
